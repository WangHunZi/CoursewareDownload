数据竞争

不同的线程同时访问同一内存，且至少有一个是写。

  • 两个内存访问在 “赛跑”，“跑赢” 的操作先执行
  • 例子：共享内存上实现的 Peterson 算法

数据竞争 (cont'd)

  • Weak memory model 允许不同观测者看到不同结果

数据竞争：例子

以下代码概括了你们遇到数据竞争的大部分情况

  • 不要笑，你们的 bug 几乎都是这两种情况的变种
// Case #1: 上错了锁
void thread1() { spin_lock(&lk1); sum++; spin_unlock(&lk1); }
void thread2() { spin_lock(&lk2); sum++; spin_unlock(&lk2); }

// Case #2: 忘记上锁
void thread1() { spin_lock(&lk1); sum++; spin_unlock(&lk1); }
void thread2() { sum++; }

为什么不要笑？

不同的线程同时访问同一内存，且至少有一个是写

  • “内存” 可以是地址空间中的任何内存
    • 可以是全部变量
    • 可以是堆区分配的变量
    • 可以是栈
  • “访问” 可以是任何代码
    • 可能发生在你的代码里
    • 可以发生在框架代码里
    • 可能是一行你没有读到过的汇编代码
    • 可能时一条 ret 指令
