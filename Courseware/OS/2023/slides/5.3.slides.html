放弃 (2)：执行顺序

例子：求和 (再次出现)

分两个线程，计算 $1+1+1+\ldots+1$ (共计 $2n$ 个 $1$)

#define N 100000000
long sum = 0;

void Tsum() { for (int i = 0; i < N; i++) sum++; }

int main() {
  create(Tsum);
  create(Tsum);
  join();
  printf("sum = %ld\n", sum);
}

  • -O2: 200000000 😱😱😱

放弃 (2)：程序的顺序执行假设

编译器对内存访问 “eventually consistent” 的处理导致共享内存作为线程同步工具的失效。

刚才的例子

  • -O1: R[eax] = sum; R[eax] += N; sum = R[eax]
  • -O2: sum += N;
  • (你的编译器也许是不同的结果)
